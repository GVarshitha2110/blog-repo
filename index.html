Summary

End-to-End CI/CD Flow for the Travel Blog

  1. Developer plans features (destinations, tips, hero), writes code, pushes to Git.
  2. CI checks out repo → npm ci → npm run build → tests (Playwright/Lighthouse).
  3. If green, CI creates a release and deploys to staging (preview URL).
  4. After UAT approval, CD promotes the same artifact to production, invalidates CDN cache.
  5. Operations monitor uptime & Web Vitals; issues/feedback go back to the Plan backlog.
